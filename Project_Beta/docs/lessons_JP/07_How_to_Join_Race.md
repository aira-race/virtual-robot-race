# 7. レースに参加する

これまでの研修で開発したアルゴリズムやAIモデルを使って、いよいよレースに参加します。
レースには「タイムアタック」と「直接対決」の2つの形式があります。

---

## 1. タイムアタック（1台走行）

まずは1台で走行し、コース2周の最速タイムを目指すタイムアタックに挑戦します。

### Step 1: アルゴリズム/AIモデルの完成
研修で開発した、あなたのルールベースアルゴリズムまたはAIモデルを完成させてください。

### Step 2: レース参加設定
`Robot1/robot_config.txt` を開き、以下の2つの項目を設定します。

#### プレイヤー名
`NAME` に、ランキングに表示されるあなたのプレイヤー名（半角英数10文字以内）を設定します。他の参加者と区別できる、わかりやすい名前が推奨されます。

```ini
# Player name (up to 10 alphanumeric characters, used for leaderboard)
NAME=Player0000
```

#### レース参加フラグ
`RACE_FLAG` を `1` に変更します。これを `1` にすると、走行結果が公式リーダーボードに登録されます。

```ini
# Race participation flag:
# 1 = Participate in race (results will be posted)
# 0 = Test Run only (no results posted)
RACE_FLAG=1
```

### Step 3: 走行
`main.py` を実行して、タイムアタックを開始します。
完走すると、タイムが自動的にリーダーボードに登録されます。

**リーダーボード:** [https://aira-race.com/](https://aira-race.com/)

---

## 2. 直接対決（2台走行）

次回の研修では、参加者同士の直接対決を行います。

### 提出方法
あなたが開発した `Robot1` フォルダ全体をZIP形式で圧縮し、指定の場所に提出してください。

**提出先:** (後日アナウンスします)

### レース形式
- 提出されたプログラムを使い、2台ずつでレースを行います。
- グリッド順は、タイムアタックの結果に基づいて決定されます。タイムの速い方が**Robot1**（前方グリッド）となります。

> **【重要】スタート制御のロバスト性**
> Robot1とRobot2では、スタートシグナルが見える位置や角度が異なります。
> どちらのグリッドからでも安定してスタートが切れるよう、ロバスト（堅牢）なスタート検出ロジックを実装しておくことが勝利の鍵です。

**Race your algorithm!**
