# 5. ルールベース制御とAIオーケストレーション

この研修では、難しそうに見えるルールベース制御に挑戦します。
ただし、Pythonプログラミングを一から書く必要はありません。**AIアシスタント（Gemini Code Assistなど）を最大限活用して課題を達成するのが、この研修の本質です。**

**学習目標:**
- このシステムが「何を」制御しているかを理解する（入力・出力・タイミング）
- AIアシスタントとのVibe Codingで制御アルゴリズムを開発・改善する手法を体験する
- 自分の「勝つための戦略」を実装し、実際に走らせる

---

## 1. このシステムは何をしているのか？（MIMO制御）

### 1.1 入力と出力

このロボットの制御システムは、**MIMO（Multiple Input, Multiple Output）**と呼ばれる構造を持っています。

| 種別 | データ | 詳細 |
|------|--------|------|
| **入力① 画像** | RGB画像 | 224×224ピクセルのJPEG（ロボット搭載カメラ映像） |
| **入力② バッテリー** | SOC | 浮動小数点値（例: `0.85`）。バッテリー残量を0.0〜1.0で表す |
| **出力① アクセル** | `drive_torque` | -1.0〜+1.0の正規化トルク |
| **出力② ハンドル** | `steer_angle` | ±0.524 rad（±30度）の操舵角 |

たった**2種類の入力**から、**2種類の出力**を決定するのが制御の仕事です。

### 1.2 何を達成するか？

**目標**: 2周を最短タイムで完走する。

**制約条件:**

| 制約 | 内容 | 違反したら |
|------|------|-----------|
| フライングしない | GOシグナル前に動かない | 即失格（FalseStart） |
| 落ちない | コース外に落下しない | 退場（Fallen） |
| バッテリーを使い切らない | SOCが0になると走行不能 | 障害物化（BatteryDepleted） |

> **補足**: 壁や他ロボットへの衝突はSOCが減少するペナルティですが、即失格にはなりません。ただし積み重なれば致命的になります。

### 1.3 タイミング

制御ループは **20fps（50ms周期）** で動作します。50msごとに画像とSOCを取得し、出力を決定して送信します。

## 2. ファイル構造とデータの流れ

### 2.1 関連ファイル

```
Robot1/
├── robot_config.txt          ← MODE_NUM=3 でルールベースモードに切替
├── rule_based_input.py       ← メインの制御ループ（ここが司令塔）
├── rule_based_algorithms/    ← アルゴリズム群（ここを書き換えて開発する）
│   ├── perception_Startsignal.py   スタート合図の検出
│   ├── sliding_windows.py          白線検出（スライディングウィンドウ法）
│   ├── driver_model.py             操舵・速度の決定ロジック
│   ├── status_Robot.py             ロボットの状態管理
│   ├── perception_Lane.py          レーン認識ユーティリティ
│   ├── perception_trackposition.py トラック上の位置推定
│   ├── Linetrace_white.py          白線トレースの実装例
│   └── debug_utils.py              デバッグ画像の生成
└── data_interactive/         ← リアルタイムデータ置き場（自動生成、変更不要）
    ├── latest_RGB_a/         ← 最新の画像ファイル（50msごとに更新）
    ├── latest_RGB_b/
    └── latest_SOC.txt        ← 最新のSOC値
```

### 2.2 データの流れ

```
Unity（シミュレーター）
  │  50msごとに更新
  ▼
data_interactive/
  latest_RGB_a or b  ← 224×224 JPEG
  latest_SOC.txt     ← "0.850" などの浮動小数点テキスト
  │
  ▼ rule_based_input.py が読み込む
update() 関数
  ├─ 画像を取得・解析 → lateral_px, theta_deg を計算
  ├─ SOC を取得
  ├─ driver_model.py が drive_torque, steer_angle を決定
  └─ グローバル変数 driveTorque, steerAngle に書き込み
  │
  ▼ main.py が呼び出す
get_latest_command()  ← 出力処理はここ
  └─ {"type":"control", "driveTorque": ..., "steerAngle": ...} を返す
  │
  ▼ WebSocket で Unity へ送信
```

### 2.3 入力を取得する処理

`rule_based_input.py` の `update()` 関数内：

```python
soc = data_manager.get_latest_soc(robot_id)        # SOC: float 0.0〜1.0
rgb_path = data_manager.get_latest_rgb_path(robot_id)  # 画像パス
pil_img = Image.open(rgb_path).convert("RGB")       # PIL画像として読み込み
```

### 2.4 出力する処理

同じく `rule_based_input.py` の `get_latest_command()` 関数：

```python
def get_latest_command():
    return {
        "type": "control",
        "robot_id": robot_id,
        "driveTorque": round(float(driveTorque), 3),
        "steerAngle": round(float(steerAngle), 3),
    }
```

`update()` がグローバル変数 `driveTorque` / `steerAngle` を更新し、`main.py` が `get_latest_command()` を呼び出してWebSocket送信します。

## 5. アルゴリズム開発：勝つための戦略を立てよう

### 5.1 戦略を考える

あなたは今、**戦うべきフィールドにAI Agentとともに立っています。**

まずAIアシスタントと対話しながら、**方針（戦略）** を決めましょう。

```
このルールベース制御システムを使って、
2周を最短タイムでゴールするための戦略を提案してください。
勝ち条件と制約条件を踏まえて、
現在のサンプルアルゴリズムのどこを改善すれば良いか教えてください。
```

考えるべきポイント例：
- もっと速く走るには？（`v_max` を上げる、カーブ減速を調整する）
- レーンを見失ったときの回復をうまくする（`hold` / `search` ロジック）
- SOCを温存しながら走るには？（`use_soc_scaling` の活用）
- スタート合図の検出精度を上げるには？

### 5.2 開発の流れ

1. **方針決定**: AIアシスタントと相談して改善方針を決める
2. **実装**: `rule_based_algorithms/` 内のファイルをAIアシスタントと一緒に書き換える
3. **テスト走行**: `main.py` を起動して実際に走らせる
4. **分析**: ターミナルログ・デバッグ画像・`metadata.csv` で結果を確認
5. **反復**: 改善してまた走らせる

あくまでサンプルプログラムなので、それを活用してもいいですが、逆にその思想にとらわれすぎないようにしましょう。AIアシスタントをうまく活用し、導くのがエンジニアの役目の一つです。

> **コツ**: サンプルのまま使ってもOKです。パラメーターを少し変えるだけでも走りが変わります。大きく書き換えるか、細かく調整するか、方針はあなた次第です。

### 5.3 主なパラメーター（`driver_model.py`）

変更しやすいパラメーターを以下に示します：

| パラメーター | 意味 | デフォルト値 |
|------------|------|------------|
| `v_max` | 最大速度（直線） | `0.55` |
| `v_min` | 最低速度 | `0.15` |
| `k_theta` | 角度への反応ゲイン | `0.45` |
| `k_lateral` | 横ズレへの反応ゲイン | `0.30` |
| `alpha_smooth` | 出力の平滑化係数（大きいほどなめらか） | `0.50` |
| `pulse_enabled` | カーブでのパルス制御を使う | `True` |
| `search_steer_const` | レーンロスト時の旋回角 [rad] | `0.6` |

### 5.4 バージョン管理を忘れずに

制御開発が始まったので、**GitHubにコミット・プッシュ** しておきましょう。
「試した版」を残しておくと、うまくいかないときに戻れます。

```bash
git add Robot1/rule_based_algorithms/
git commit -m "Rule-based: v1.0 - 最初の戦略"
git push
```
